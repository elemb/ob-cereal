#+TITLE: ob-cereal

* ob-cereal

Org-babel support for talking to a serial REPL from inside Org source blocks.
Write an experiment in a =cereal= block, evaluate it, and the device's reply
lands in the buffer as results. Built for interactive Forth systems such as
[[https://github.com/tabemann/zeptoforth][zeptoforth]] and Mecrisp, but it will
talk to anything that speaks over a serial port.

This is a fork of =ob-uart= by Andreas Müller; the original idea and the bulk of
the design are his.

* Status

Working. Each block evaluation is a single, self-contained transaction: open the
port, send the block, read replies until a timeout, close the port. There is no
persistent session — see [[*Caveats][Caveats]].

* Requirements

- Emacs with Org
- A serial device exposing a REPL (e.g. a Raspberry Pi Pico running a =full_usb=
  zeptoforth image at 115200 baud, appearing as =/dev/ttyACM0= or
  =/dev/cu.usbmodem*=)

* Install

Put =ob-cereal.el= on your =load-path= and require it:

#+BEGIN_SRC emacs-lisp
(add-to-list 'load-path "/path/to/ob-cereal")
(require 'ob-cereal)
#+END_SRC

Requiring the file defines =org-babel-execute:cereal=, which is all Org needs to
evaluate =cereal= blocks. If your setup gates languages through
=org-babel-load-languages=, add =(cereal . t)= there as well.

* Usage

Set the port and speed as header arguments; put REPL input in the body:

#+BEGIN_SRC org
,#+BEGIN_SRC cereal :port /dev/cu.usbmodem31401 :speed 115200
armv6m import
: interp-delay ( -- )
  [inlined]
  code[
  $E7FF h,
  $E7FF h,
  $E7FF h,
  ]code
;
see interp-delay
,#+END_SRC
#+END_SRC

Evaluating the block sends those lines to the device and captures the reply,
including zeptoforth's disassembly of the word you just defined:

#+BEGIN_EXAMPLE
20018E36 B500:      interp-delay:         PUSH {LR}
20018E38 E7FF:                            B 1F <$20018E3A>
20018E3A E7FF:      1:                    B 2F <$20018E3C>
20018E3C E7FF:      2:                    B 3F <$20018E3E>
20018E3E BD00:      3:                    POP {PC}
 ok
#+END_EXAMPLE

* Header arguments

| Argument       | Default        | Meaning                                              |
|----------------+----------------+------------------------------------------------------|
| =:port=        | =/dev/ttyUSB0= | Serial device path                                   |
| =:speed=       | =115200=       | Baud rate                                            |
| =:bytesize=    | =8=            | Data bits                                            |
| =:parity=      | =nil=          | =odd= or =even=; anything else means no parity       |
| =:stopbits=    | =1=            | Stop bits                                            |
| =:flowcontrol= | =nil=          | =hw= or =sw=; anything else means none               |
| =:timeout=     | =2=            | Seconds to read replies before closing               |
| =:lineend=     | =\n=           | Appended to the body before sending                  |
| =:ienc=        | =raw=          | =hex=: read the body as whitespace-separated hex     |
|                |                | bytes and send the raw bytes                         |
| =:oenc=        | =raw=          | =hex= / =HEX=: return received bytes as hex          |

Set project-wide defaults with =org-babel-default-header-args:cereal=.

* Caveats

- *No persistent session.* Every evaluation opens and closes the port. Whether
  the /device's/ state survives between blocks is up to the device: opening a
  USB-CDC port on an RP2040 does not reset it, so definitions persist — but that
  is the target's behavior, not something ob-cereal maintains.
- *Every block waits out =:timeout=.* The read loop has no early exit on a
  prompt, so a block takes roughly =:timeout= seconds regardless of how quickly
  the reply arrives. Lower =:timeout= for snappier interaction; raise it for slow
  or chatty operations.
- *Control characters are stripped* from results; newlines and tabs are kept.
- *A numeric argument that parses to 0 falls back to its default* (=:timeout 0=
  becomes 2). This is a quirk of the number coercion, not an intended feature.
